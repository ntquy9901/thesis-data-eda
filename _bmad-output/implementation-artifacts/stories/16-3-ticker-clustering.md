# Story 16.3 — Ticker Clustering + Entity Embeddings

**Epic:** 16 (Advanced News Signal with Embedding + Ticker-Specific Models)
**Story key:** `16-3-ticker-clustering`
**Status:** backlog

## Context

Only ~25% of tickers (7-8 out of 30 VN30 constituents) show positive ΔR² from news features. The remaining 75% see no improvement or degradation when news features are added to the price-only baseline. This suggests a one-size-fits-all modeling approach is suboptimal — news-aware models should only activate for tickers where news actually helps, and tickers with similar news-sensitivity profiles should be grouped.

The existing `significance.json` output from Epic 9 already contains per-ticker ΔR² values for each prediction horizon. This story clusters tickers based on their news-sensitivity profile and creates ticker-level entity embeddings for the Ridge pipeline.

## Requirements (Acceptance Criteria)

- [ ] New file: `src/modeling/ticker_clusters.py`
- [ ] `compute_news_sensitivity()`: loads `eda_output/significance/significance.json`, extracts per-ticker ΔR² for `pk_t+10`

